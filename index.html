The
Cognitive
Gap.

Traditional finance apps are spreadsheets in disguise. They report history. They never interrupt the pattern driving it — and that is the gap Pulse was built to close.

02 // The Solution

What
Pulse
Is.

Pulse-AI is a behavioral finance engine — not a budgeting app. It detects your financial rhythm, maps it to your behavioral patterns, and deploys Nova to interrupt the cycle before capital loss occurs.

Pulse operates at the intersection of real-time transaction telemetry, machine learning, and behavioral psychology. Every transaction that flows through Plaid's read-only API is processed by Pulse's signal layer — categorized, weighted by context, and fed into a proprietary Stress Index that correlates your spending velocity with identifiable behavioral triggers. The output is not a pie chart. It is a live behavioral profile — and it improves every day you use it.

Spending Trigger Detection
Pulse identifies emotional and contextual patterns behind your transactions — not just the amounts, but the WHY driving them. Stress-spend Thursdays. Impulse Fridays. Identified.
Financial Stress Index
A proprietary score that correlates your behavioral data with cash flow patterns, transaction velocity, and category drift — updated in real time.

Live Plaid Telemetry

04 // The Architecture

Technical
Heart
beat.

Pulse is engineered for deterministic data integrity at every layer of the stack. From the Plaid handshake to the PostgreSQL behavioral pipeline, every component was selected to maximize signal fidelity and minimize latency. No third-party aggregators. No intermediary data brokers. Your financial data flows directly into a row-isolated, JWT-secured Supabase/PostgreSQL instance and never leaves it unencrypted.

Layer 01
Zero-Knowledge Bank Access
Plaid's read-only API forms the data ingestion layer. Pulse never sees credentials, never initiates transactions, and has no write access to any financial account. The bank handshake is a one-time token exchange — Pulse receives transaction signals, not bank access.
Layer 02
Real-Time Signal Processor
Every transaction is passed through a rule-based categorization engine with high-confidence auto-assignment. The engine runs on FastAPI with Motor (Async MongoDB) fallback for sub-millisecond categorization under heavy load. Star Schema normalization ensures the behavioral pipeline never degrades under scale.
Layer 03
Behavioral Pipeline (Python / Pandas)
Categorized transactions flow into a Python/Pandas behavioral analysis layer. This is where the Stress Index, spending trigger detection, and 90-day predictive modeling operate. All computations are deterministic — no black boxes, no unexplained scores.
Layer 04
PostgreSQL Star Schema (Supabase)
The behavioral output is persisted in a high-fidelity Star Schema: dim_users, fact_transactions, dim_categories, and behavioral event tables. Row-Level Security (RLS) policies enforce owner-only data access at the database layer — not just the API layer.
Layer 05
Nova AI Layer (OpenAI + Behavioral Context)
Nova's intelligence is powered by OpenAI's foundation model, fine-tuned with Pulse's behavioral context window — your stress index, spending triggers, historical patterns, and active goals. Each session is context-aware. Each response is grounded in your actual data, not generic advice.
Layer 06
Encrypted Receipt Vault (AWS S3)
Receipt uploads are encrypted in transit via TLS and stored at rest with AES-256 in a personal S3 bucket isolated to your account. Pre-signed URLs provide sub-millisecond access without exposing the underlying storage path. Full audit trail for tax export.
